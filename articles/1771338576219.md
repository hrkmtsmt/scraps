## SQLのSELECT文の定数出力はいつ使えるのか

```sql
SELECT '定数' AS CONSTANT, id FROM products;
```

本を読んでいて、SELCT文の定数出力の説明があった。
そもそも、こんな機能初めて知ったので、こんなも機能もあるのかと驚いた。
と、同時にこの機能いつ使うのか想像できなかった。

複数のデータを引っ張ってくるとき、全てのデータに対して同じデータを出力するケースてなんだとなった。

想像つかなかっとので、Claludeに問うてみて、いくつかしっくりくる答えがあった。

ひとつめは、デバッグ目的の定数。以下のようなSQLを実行すると、ログにデバッグで実行したSQLだとひと目でわかるので、使えそうだと思った。

```sql
SELECT 'debug' AS debug, id FROM products;
```

ふたつめは、データ移行時。SELECT文で得た結果に加えて特定のデータを付け足して、他のテーブルにインサートしたいときに使えそうだと思った。
以下のように、商品テーブルからログテーブル一括でデータを挿入した時など便利そう。

```sql
INSERT INTO log SELECT '2026-01-01' created_at, id FROM products;
```
